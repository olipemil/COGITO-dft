# File Structure

As LLMs become better at accurately parsing large codebases, documentation has become less 'important' as many people turn to LLMs to know how to use the code. 
In my experiance, this has made gaining a intutive sense for code structure much harder. Then, when you want just one thing from a complicated codebase (or heaven forbid nosense bugs happen), things get messy and LLMs often solve nonoptimally.
Even they do fine, I just want humans to understand what is going on as best a possible (because its cool).
<br>
So! I have created this little interactive visualization of different file inputs and outputs of my different COGITO modules. I hope you enjoy clicked through them and this proves to be a handy reference if ever you want to make (or skip) some file. 
<br>
The boxes are bolded if the input/output is required. The boxes are colored red for quality accessment files and yellow for debug/developer files. More tags and colors will added when I (hopefully) get a bit of time for it. This is still a bit underdevelopment and as such there may be small errors here and there.
